How it works

Cowx is a static site hosted on GitHub Pages. There is no application server. Weather data is pre-fetched by a GitHub Action about every 45 minutes and written into JSON files under public/data/. Your browser loads only the location you need.

Finding your location

  1. If you previously viewed or favorited a site, we offer that first (stored only in your browser).
  2. You can search by city/county name or Colorado ZIP code.
  3. The Locate button uses the browser Geolocation API (permission required).
  4. If needed, a coarse IP-based location is requested from a public geolocation API in your browser, then snapped to the nearest Colorado catalog site. Out-of-state visitors still get the nearest CO site.

Data pipeline

The pnpm fetch orchestrator calls independent adapters (Open-Meteo, NWS, CoAgMET, Aviation Weather, optional PurpleAir and AirNow). Each source can succeed or fail on its own. Results are merged into:

  • index.json — slim list for search and the map
  • locations/{slug}.json — full drill-down payload
  • alerts.geojson — warning polygons when geometry is available
  • meta.json — build time and per-source status (ok / partial / error / skipped)
  • co-zips.json — ZIP centroids for search

If a non-critical source fails, the site still publishes. If critical sources all fail, the Action fails and operators are notified.

Dashboards

Location pages show the full detail set we collect for Colorado audiences — current conditions, hourly and multi-day forecasts, alerts, agricultural stations, aviation METARs/TAFs, and air quality — all on one page (not filtered by role). Citizen, pilot, farmer, and firefighter needs shape what we fetch, not which panels you can open. Offsite links always point back to authoritative providers.

Failure notifications

When the scheduled update workflow fails, GitHub Actions can post to a Discord webhook configured as the repository secret NOTIFY_WEBHOOK_URL, and open or update a GitHub Issue with a link to the failed run. Secret values are never committed to the repository.

Privacy

  • No accounts and no cowx backend database.
  • Favorites use localStorage keys cowx:favorites and cowx:lastLocation only.
  • No third-party analytics by default.
  • API keys for PurpleAir/AirNow are used only in GitHub Actions at build time — never in the browser.
